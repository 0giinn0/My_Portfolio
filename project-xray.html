Concept

X-Ray explores two canvas-based reveal techniques. Both use stacked canvases with CSS masking to expose a hidden layer beneath a visible surface — one with a persistent trail, one with instant cursor-following.

Trail Mask — How It Works

Three stacked canvases: a base layer, a reveal layer, and an offscreen mask canvas. The mask canvas draws radial gradients along the cursor trail — newer points are more opaque. This canvas is converted to a data URL and applied as a CSS mask-image on the reveal layer in real-time.

Hover Reveal — How It Works

Simpler approach: two canvases stacked. The reveal layer uses a pure CSS radial-gradient mask with var(--mx) / var(--my) custom properties updated on mousemove. The gradient's circle radius is controlled by a slider. No JavaScript mask compositing — just CSS variables driving a mask-image.

Use Cases

Portfolio hero sections — replace static images with an interactive reveal that invites exploration. Visitors naturally want to "peek" behind the surface, increasing engagement time.

Product showcases — show a product's exterior on the base layer and its internal components, wireframes, or technical specs on the reveal layer. Customers can X-ray through the surface to understand what's inside.

Data visualization — layer raw data beneath a cleaned-up chart. Analysts can hover to drill down from summary to detail without navigating away.

Educational content — anatomy, architecture, engineering diagrams. The trail mask lets students explore at their own pace, revealing layers progressively rather than all at once.

Before/after comparisons — show the "after" as the base and the "before" as the reveal. The cursor becomes a lens that lets users compare states in the same viewport.

Game UI / storytelling — hide clues, maps, or narrative elements beneath a surface layer. Players discover content by moving through the scene, creating a sense of agency and discovery.

Implementation Notes

Performance — the Trail Mask converts the mask canvas to a data URL every frame. For production, consider using canvas.toBlob() with URL.createObjectURL() instead of toDataURL() to avoid base64 encoding overhead. The Hover Reveal is essentially free — it's a single CSS property update per frame.

Mobile — both effects work on touch devices. On mobile, the trail mask responds to touchmove events and the hover reveal follows finger position. Consider increasing brush radius for touch input since finger precision is lower than mouse.

Accessibility — the effects are decorative. The base layer content should still be readable without interaction. Add a prefers-reduced-motion check to disable the trail animation for users who are sensitive to motion.
